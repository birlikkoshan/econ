# Техническая документация проекта

**Корпоративный сайт ТОО «Евразийский консалтинговый консорциум» (ECON)**

---

## 1. Назначение и общее описание

Проект представляет собой корпоративный сайт-визитку компании, выступающей
уполномоченным изготовителем лицом на территории Республики Казахстан и ЕАЭС.
Задача сайта — за несколько секунд объяснить, чем занимается компания, показать
её услуги, продемонстрировать юридическую надёжность и дать простой способ
связаться.

Сайт реализован как многостраничное одностраничное приложение (SPA) на React с
клиентской маршрутизацией, мультиязычным интерфейсом (4 локали) и серверной
функцией для приёма заявок с формы обратной связи.

Адрес развёртывания: <https://eccllp.vercel.app>

## 3. Архитектура проекта

### 3.1 Структура каталогов

```
api/                      → Serverless-функции Vercel
  contact.ts              → Приём и отправка заявок по SMTP (nodemailer)

src/
  app/                    → Точка сборки приложения
    App.tsx, router.tsx   → Корень и маршруты
    providers/            → Глобальные провайдеры (i18n и пр.)
  pages/                  → Страницы (Home, About, Services, Legislation,
                            Partners, Contacts)
  components/
    layout/               → Header, Footer, MainLayout, фон-орнамент
    ui/                   → Переиспользуемые элементы (Button, Input, Section,
                            Accordion, Reveal, PageHero, Counter, LanguageSwitcher…)
  features/
    contact/              → Форма обратной связи (ContactForm, useContactForm,
                            submitContactForm)
  constants/              → Конфигурация и контент (site, navigation, content)
  lib/                    → Утилиты (validation, useReveal, utils)
  i18n/                   → Инициализация i18next
  locales/                → Файлы переводов (ru, en, kk, kk-Latn)
  types/                  → Общие типы
```

### 3.2 Слои и поток данных

`страница (pages) → секции и UI-компоненты (components) → данные (constants) + тексты (i18n)`

Разделение принципиальное: в `constants/` хранятся **языконезависимые данные**
(реквизиты, ссылки, список партнёров, иконки), а все **переводимые подписи**
берутся из `locales/` по ключам i18n. Это позволяет менять язык интерфейса без
дублирования структурных данных.

## 4. Ключевые подсистемы

### 4.1 Маршрутизация

`src/app/router.tsx` описывает шесть основных маршрутов внутри `MainLayout`:
`/`, `/about`, `/services`, `/legislation`, `/partners`, `/contacts`.
Несуществующие пути перенаправляются на главную (`*` → `/`). Компонент
`ScrollToTop` сбрасывает прокрутку наверх при смене страницы.

На стороне хостинга `vercel.json` настраивает SPA-fallback: все запросы, кроме
`/api/*`, отдают `index.html`, что обеспечивает работу клиентской маршрутизации
при прямом переходе по ссылке.

### 4.2 Мультиязычность (i18n)

Поддерживаются четыре локали: **русский (по умолчанию)**, английский, казахский
(кириллица) и казахский (латиница, `kk-Latn`). Инициализация — в
`src/i18n/index.ts`. Помимо переключения текстов, при смене языка синхронизируются
атрибуты `<html lang>` и `data-script`, что позволяет CSS подменять шрифт для
казахских символов (ә, ғ, қ, ң, ұ), отсутствующих в основном шрифте Manrope.

Переключатель письменности казахского (Кирил / Latyn) реализован как подрежим
языка «kk».

### 4.3 Форма обратной связи

Состоит из трёх частей:

- **UI** — `features/contact/ContactForm.tsx`: поля «имя», «компания», «email»,
  «сообщение» с выводом ошибок рядом с полями и сообщением об успехе.
- **Логика** — `useContactForm.ts`: управление состоянием, клиентская валидация
  (`lib/validation.ts`), обработка статусов отправки и ошибок.
- **Отправка** — `submitContactForm.ts`: `POST /api/contact`, разбор кода ошибки
  из ответа (класс `ContactSubmitError`).

Серверная часть — `api/contact.ts` (Vercel serverless): принимает только `POST`,
повторно валидирует данные, читает SMTP-учётные данные из переменных окружения и
отправляет письмо через Gmail SMTP с помощью nodemailer. В письме проставляется
`replyTo` адреса отправителя, что позволяет отвечать напрямую.

Переменные окружения:

| Переменная | Назначение |
| --- | --- |
| `SMTP_USER` | Логин почтового ящика (Gmail) |
| `SMTP_APP_PASSWORD` | Пароль приложения Gmail |
| `CONTACT_TO` | Получатель заявок (по умолчанию `econ2007kz@gmail.com`) |

Коды ответа: `200 ok`, `400` (ошибка валидации), `405` (метод не POST),
`503 mail_not_configured`, `500 send_failed`.

### 4.4 Анимации и адаптивность

Лёгкие анимации появления секций при прокрутке реализованы через хук `useReveal`
(на базе `IntersectionObserver`) и компонент-обёртку `Reveal`. Вёрстка построена
по принципу mobile-first на брейкпоинтах Tailwind и корректно отображается как на
телефоне, так и на широком экране. Дополнительные элементы оформления: счётчик
(`Counter`), фоновый казахский орнамент (`KazakhOrnamentBackground`), карусель
логотипов партнёров.

## 5. Сборка и развёртывание

Production-сборка выполняется командой `npm run build`: сначала проверка типов
(`tsc -b`), затем сборка оптимизированного бандла Vite в каталог `dist/`.
Хостинг — **Vercel**: статика раздаётся из `dist/`, serverless-функция из `api/`
доступна по пути `/api/contact`. Доменное имя по умолчанию — `eccllp.vercel.app`
(может быть заменено на собственный домен компании).

## 6. Соглашения по коду

- Алиас `@` указывает на `src/` (настроен в `vite.config.ts` и `tsconfig`).
- Комментарии в коде — на русском, поясняют **почему**, а не **что**.
- Контент и данные вынесены из компонентов в `constants/` и `locales/`,
  компоненты остаются презентационными.
- Линтинг — oxlint; типизация строгая (TypeScript).
