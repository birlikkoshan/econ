# Инструкция по разработке сайта «Евразийский консалтинговый консорциум»

> Этот файл — главный ориентир для работы. Читать в начале каждой сессии (особенно после очистки контекста).

## Главное правило работы

**Делаем всё по одному. 1 страница за 1 промпт.**
Не пытаться сверстать весь сайт сразу — пользователь даёт страницы по очереди.
После завершения страницы — отметить её в разделе «Статус страниц» ниже.

## О компании

- **Название:** ТОО «Евразийский консалтинговый консорциум» (ЕКК / ECON).
- **Суть бизнеса:** Уполномоченное изготовителем лицо в ЕАЭС, сертификация ТР ЕАЭС,
  декларирование соответствия, техническая документация, маркировка EAC, испытания продукции.
- **Аудитория:** B2B, в т.ч. иностранные производители → сайт должен показывать
  юридическую надёжность и экспертизу в ТР ЕАЭС.
- **Логотип:** `src/assets/logo.png` — чёрно-белый глобус с надписью ECON.

## Дизайн

- **Эталон стиля:** `docs/Вариант 1 - Изумруд (автономный).html` — выбранный вариант.
- **Главный цвет:** изумрудно-зелёный `#0f5136` (фон/акцент), белый, светлые поверхности.
- ⚠️ Текущий `src/styles/index.css` содержит amber/gold токены из шаблона —
  привести палитру к изумрудно-зелёной из эталона при работе над страницами.

## Технический стек (уже инициализирован)

- React 19 + TypeScript + Vite 8
- Tailwind CSS 4 (`@tailwindcss/vite`, токены в `src/styles/index.css`)
- react-router-dom 7 — роутинг в `src/app/router.tsx`
- i18n: react-i18next, языки `ru` (по умолч.), `en`, `kk` — `src/locales/*.json`
- Деплой: Vercel (`vercel.json`)
- Команды: `npm run dev`, `npm run build`, `npm run lint`

## Структура проекта

```
src/
  app/        App.tsx, router.tsx
  pages/      страницы (HomePage.tsx ...)
  sections/   секции страниц (Hero, About, Services, Legislation, Partners, Contact)
  components/  layout/MainLayout, общие UI
  constants/  navigation.ts, site.ts
  locales/    ru.json, en.json, kk.json
  styles/     index.css (Tailwind + тема)
```

Навигация (`src/constants/navigation.ts`): home, about, services, legislation, partners, contact.

**Формат сайта (решено):** multi-page — отдельные страницы с маршрутами
(`/`, `/about`, `/services`, `/legislation`, `/partners`, `/contacts`).
Текущий скаффолд — одностраничник с якорными секциями; переводим на роутинг
по страницам через `react-router-dom` в `src/app/router.tsx` + общий `MainLayout`.
